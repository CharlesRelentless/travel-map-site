# 新藏线 G219 + 阿里 + 珠峰 7 日交互地图

这是一个面向手机和桌面浏览器的单页旅行辅助地图。项目目标不是替代专业导航，而是在一张轻量网页里集中展示 2026 年 6 月 6 日开始的新藏线 G219、阿里、珠峰 7 日行程，包括分日路线、景点说明、海拔、预计到达时间、到达时段天气风险、实时定位和下一目的地粗略 ETA。

## 最开始的设定

项目最初设定为一个静态 HTML 页面：

- 行程主题：新藏线 G219 + 阿里 + 珠峰 7 日自驾路线。
- 旅行起点：喀什方向出发，终点到拉萨。
- 核心视图：左侧行程列表，右侧交互地图。
- 路线组织：按 Day 1 到 Day 7 分组，每一天有独立颜色、路线线段和景点 marker。
- 点位信息：每个站点包含名称、经纬度、参考海拔、计划到达时间和说明。
- 天气目标：按站点经纬度请求 Open-Meteo 天气数据，在弹窗里展示温度曲线。
- 地图目标：优先使用 OpenStreetMap 相关底图，适合部署为无需后端的轻量网页。

## 运行方式

这是一个静态网页项目，核心文件是：

```text
index-amap.html
```

桌面端通过本地代理访问 `http://127.0.0.1:8000/index-amap.html`。

手机端建议通过 HTTPS 部署后访问，因为浏览器定位能力通常要求安全上下文。直接用 `file://` 或普通 HTTP 打开时，部分手机浏览器可能拒绝定位权限。

半离线地图瓦片缓存同样建议通过 HTTPS 或 localhost 使用。直接打开 HTML 文件时，Service Worker 无法注册，只能依赖浏览器普通缓存。

## 数据源和外部依赖

当前页面依赖：

- Leaflet：地图渲染。
- OpenStreetMap 相关瓦片源：地图底图。
- Open-Meteo：天气预报、逐小时降水和风速数据。
- 浏览器 Geolocation API：实时定位。

项目不包含后端服务，也没有 API key。

## 准确性和限制

需要明确的限制：

- 路线点位是第一版近似坐标，不等于精确导航路线。
- 天气来自数值天气预报，不等于雷达临近预报，也不等于官方灾害预警。
- 高原和山区天气变化快，极端天气判断必须结合当地气象预警、道路管制、边防要求和实时导航。
- 当前 ETA 是粗略估算，不能用于精确行车安排。
- OSM 在线瓦片和 Open-Meteo 均依赖网络，国内移动网络环境下仍可能波动。
- 半离线瓦片缓存只覆盖已经浏览或主动缓存过的区域，不能替代完整离线地图。
- 浏览器可能在存储压力较大时清理 Cache Storage 或 localStorage。
- 当前高德 Web 服务 Key 写在前端，公开发布会泄露 Key。

## 后续可升级方向

优先级较高的后续方向：

- 将高德 Web 服务 Key 移到后端代理，避免前端泄露。
- 增加 API key 后端代理，避免把 key 暴露在前端。
- 接入完整离线地图包或自托管瓦片，进一步提升无人区可用性。
- 增加路线偏离提示。
- 增加缓存管理面板，显示已缓存瓦片数量和天气缓存时间。

## 2026-06-06 墨迹天气与极端天气预警阶段

- 达坂、湖泊、珠峰等无人区继续使用 Open-Meteo 经纬度预报，避免直接套用附近县城温度。

- 墨迹响应按 CityID 缓存 20 分钟，减少套餐调用次数。
- 墨迹失败时自动回退到 Open-Meteo；两者都失败时显示本地估算并明确标注。
- 城市站点同时显示墨迹当前实况温度和计划到达时刻预报温度；两者含义不同，预报值未变化不代表接口没有刷新。
- 无人区站点不直接使用附近县城的墨迹当前温度，避免产生误导。

```powershell
conda run --no-capture-output python weather_proxy.py
```

电脑访问：

```text
http://127.0.0.1:8000/index-amap.html
```

同一局域网手机访问时，将 `127.0.0.1` 替换为电脑局域网 IP。GitHub Pages 只能部署静态文件，无法运行 `weather_proxy.py`；若需要公网手机访问墨迹天气，应将代理部署到云函数或服务器，并继续将 AppCode 保存为服务端环境变量。

### 公网 HTTPS 部署

项目提供 `render.yaml`，用于将地图页面和墨迹代理部署为同一个 Render HTTPS Web Service。手机在任何网络中直接访问 Render 分配的 HTTPS 地址，不再依赖电脑局域网。

部署步骤：

1. 将项目推送到私有 Git 仓库。
2. 在 Render 中通过 Blueprint 创建服务并选择该仓库。
3. 设置服务端环境变量 `MOJI_APPCODE`，不要将真实值提交到仓库。
4. 部署完成后访问 `https://<服务域名>/index-amap.html`。
5. 在高德开放平台 Web 端 Key 的域名白名单中加入新的 Render HTTPS 域名。

生产环境建议使用不会休眠的常驻实例。极端天气预警不应依赖会自动休眠、首次访问需要冷启动的免费实例。

公网服务安全限制：

- 只公开 `index-amap.html`、`sw.js`、`/healthz` 和 `/api/moji`。
- 墨迹接口只允许行程中已配置的 CityID。
- 每个来源地址 5 分钟最多请求 90 次。
- 墨迹临时失败时，可使用最多 6 小时内的旧缓存并标记为 stale。
- 若地图与代理分开部署，设置 `TRAVEL_MAP_ALLOWED_ORIGIN` 为地图的完整 HTTPS 来源。

### 2026-06-06 前方阶段恶劣天气提醒
